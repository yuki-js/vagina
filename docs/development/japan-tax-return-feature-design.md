# 日本の確定申告（個人）機能 拡張設計書

**対象プロダクト:** VAGINA（Voice AGI Notepad Agent）  
**作成日:** 2026-03-06  
**目的:** 日本の確定申告・決算作業を支援するための「検討・設計」と、並列実行可能なサブタスク計画を定義する。

---

## 1. 設計方針（要約）

- 本機能は **申告そのものを自動提出する** のではなく、まずは **申告作業を安全に半自動化する支援レイヤー** として設計する。
- 法令・様式改正に追従するため、税制ロジックは **年度バージョン管理** を前提に分離する。
- 音声エージェント + ノート + スプレッドシート + ツール実行基盤を活用し、ユーザー作業を段階的に短縮する。
- 個人の所得税確定申告を中心に、将来的な消費税・法人決算対応へ拡張可能なドメインモデルを採用する。

---

## 2. Deep Research の調査軸（日本ドメイン）

以下を優先ソースとして調査・継続監視する:

1. 国税庁（NTA）確定申告特集、タックスアンサー、各種様式・手引き
2. e-Tax公式サイト（利用制限、仕様書更新、受付時期）
3. 電子帳簿保存法・インボイス制度・改正税法の年次更新
4. 地方税（住民税）連携に必要な提出・控除情報

設計上の原則:

- ルール根拠（条文/番号/公開URL）をメタデータとして保持
- 「推論結果」と「根拠」を同時提示し、監査証跡を残す
- 法改正の施行日・適用開始日ベースで判定する

## 3. ターゲット機能（てんこ盛り版）

### 3.1 申告準備オーケストレーター

- 申告年・家族構成・所得区分・帳簿方式（白色/青色）ヒアリング
- 必要書類チェックリスト生成（不足書類アラート付き）
- 期限前リマインド（提出/納付/振替納税）

### 3.2 所得データ集約

- 給与所得（源泉徴収票OCR/手入力/CSV）
- 事業所得（売上・経費・家事按分・減価償却）
- 不動産所得（収入・必要経費・空室期間）
- 配当/譲渡/雑所得（証券会社明細取り込み）
- 暗号資産損益（取引履歴正規化、移動平均/総平均などの計算モード）

### 3.3 控除・税額控除エンジン

- 基礎/配偶者/扶養/障害者/寡婦（ひとり親）控除
- 社会保険料・生命保険料・地震保険料・小規模企業共済
- 医療費控除/セルフメディケーション税制
- 寄附金控除（ふるさと納税含む）
- 住宅ローン控除（年次要件と残高証明連動）

### 3.4 青色申告・決算支援

- 仕訳インポート（CSV/将来API連携）
- 勘定科目マッピング支援
- 青色申告決算書の下書き生成
- 貸借対照表/損益計算書チェック（整合性検証）

### 3.5 申告書ドラフト生成支援

- 申告書B・収支内訳書・青色申告決算書など必要帳票の下書きデータ生成
- 数値の出所トレース（どの証憑/会話/計算から作られたか）
- 提出前の矛盾検知（例: 所得0なのに社会保険料控除が過大）

### 3.6 リスク検知・説明機能

- 期限後申告リスク、加算税/延滞税の説明
- 監査で確認されやすい論点（家事按分率、交際費、現金売上）の注意喚起
- 「税務相談推奨」判定（高リスク時）

## 4. 既存アーキテクチャへの実装マッピング

本リポジトリの既存資産を活かす実装方針:

- `lib/tools/builtin/*` に税務ユースケース向けツールを追加
  - 例: `tax_rule_lookup`, `tax_deduction_eligibility_check`, `tax_return_draft_builder`
- `notepad` と `spreadsheet` を申告ワークスペース化
  - ノート: 根拠と説明文
  - 表: 収入・経費・控除の集計
- `text_agent` を深掘り推論と論点整理に利用
- `memory_*` ツールで会話ベースの申告前提条件を安全保存

## 6. 機能優先度（MVP → 拡張）

### MVP（最初の確実な価値）

1. 申告準備ウィザード
2. 給与 + 事業所得の集計補助
3. 主要控除チェック（社会保険・医療費・寄附）
4. 申告書ドラフト用の集計出力（提出そのものは人間最終確認）
5. リスク説明（期限・不足資料・矛盾）

### Phase 2

- 青色申告決算書支援の拡充
- 取込形式拡大（証券/暗号資産）
- 税制改正差分の自動ハイライト

### Phase 3

- e-Tax提出支援の高度化
- 住民税・個人事業税への横展開

---

## 7. 非機能要件

- **正確性**: 金額計算はすべて再計算可能で決定論的であること
- **追跡可能性**: すべての算出値に根拠リンクを持つこと
- **セキュリティ**: PII暗号化、アクセス制御、最小権限
- **可用性**: 法改正時も旧年度計算を継続可能
- **説明責任**: AI提案は必ず「根拠」と「要確認事項」を併記

---

## 8. リスクと回避策

| リスク | 内容 | 回避策 |
|---|---|---|
| 法改正追従遅延 | 年度更新が遅れると誤計算リスク | ルールの年度分離 + 月次更新運用 |
| データ欠損 | 証憑不足で計算不能 | 不足項目検知と入力ガイド |
| 過信リスク | AI提案を鵜呑みにする利用者行動 | 最終確認ステップを強制 |
| PII漏えい | 税務データの高機微性 | 暗号化・マスキング・監査ログ |

1. ドメインモデル確定（T07/T08/T09）

3. 申告ウィザードと不足検知（T18/T19）

5. QAマトリクスと回帰基盤（T29/T30）

---

## 10. 重要な注意事項

- 本設計は税務作業を支援するためのものであり、最終申告責任は利用者にある。
- 実装時は各年度の法令・通達・公式資料に基づいてルール定義を更新する。
- 高リスクケース（国外所得、特殊控除、修正申告等）は税理士等への相談導線を提供する。
